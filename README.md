## 1. 项目概述

本项目是一个**基于进化多任务优化（Evolutionary Multitask Optimization, EMTO）的三维点云配准特征选择系统**，使用 MATLAB 编写。

### 核心问题

在三维点云配准（Point Cloud Registration）中，常使用多种局部几何特征描述子（如 FPFH、SHOT、SpinImage、ROPS）来描述点云中每个点的局部几何结构。这些描述子拼接后形成高维特征向量（例如 672 维），其中包含大量冗余和无关特征。本系统的核心目标是：

> **从高维拼接特征中自动选择出最优的特征子集，使得点云配准（匹配）的精度最高。**

### 技术路线

1. **特征提取**：从点云中提取 FPFH、SHOT、SpinImage、ROPS 四种描述子
2. **特征重要性评估**：使用皮尔逊相关系数、互信息等方法评估每维特征的重要性
3. **多任务优化**：将特征选择问题建模为多任务优化问题，其中不同任务对应不同的特征搜索空间
4. **进化搜索**：使用遗传算法（GA）进行特征子集搜索，通过知识迁移加速收敛
5. **代理模型**：使用下采样点云作为代理模型降低评估成本
6. **对比实验**：与 ReliefF、皮尔逊相关系数、互信息、CUS-PSO、MF-CSO 等方法进行对比

## 3. 核心数据类（Public/）

### 3.1 PARAMETER 类（`Public/PARAMETER.m`）

全局参数容器，存储所有实验配置和数据。

| 属性 | 类型 | 说明 |
|------|------|------|
| `Maxnum` | 整数 | 点云数量（默认 6） |
| `yuzhi` | 浮点数 | 匹配阈值，由 `cal_yuzhi` 根据点云分辨率自动计算 |
| `dim` | 整数 | 特征总维度（各描述子维度之和，如 672） |
| `LEN` | 向量 | 每个描述子的维度，如 `[33, 352, 153, 135]` |
| `T` | cell 数组 | 各点云的仿射变换矩阵（`affine3d` 对象） |
| `p` / `pp` | 点云数组 | `p` 为下采样两次的点云（代理模型用），`pp` 为下采样一次的点云 |
| `F` / `FF` | cell 数组 | `F` 对应 `p` 的特征，`FF` 对应 `pp` 的特征 |
| `data` | cell 数组 | 原始点云数据 |
| `GrtT` / `GrtR` | cell 数组 | 真值平移向量 / 旋转矩阵 |
| `st` | 向量 | 特征的标准差（用于归一化） |
| `overlap` | 矩阵 | 点云间重叠率矩阵 |
| `dataname` | 字符串 | 数据集名称（如 `"WHU"`） |
| `name` | 字符串 | 模型名称（如 `"Campus"`） |

**初始化流程（`initPARAMETER`）**：
1. 从 `.mat` 文件加载点云数据、真值变换
2. 加载下采样点云和对应特征
3. 计算匹配阈值 `yuzhi`
4. 加载描述子维度信息 `LEN`
5. 构建仿射变换矩阵
6. 计算所有点云对之间的重叠率矩阵

### 3.2 DB 类（`Public/DB.m`）

数据库类，管理匹配对和特征重要性，同时负责任务生成。

| 属性 | 说明 |
|------|------|
| `Pair` | 真实匹配对矩阵，每行为 `[点云i, 点云j, 点i索引, 点j索引, 标签]` |
| `FI` | 特征重要性向量（1 × dim），值越大表示该维特征越重要 |

**核心方法**：

- **`initDB(object, prm)`**：初始化数据库
  1. 为每个描述子单独生成特征子集（每次仅启用一个描述子的所有维度）
  2. 使用真实匹配方法（`prm.pp` 和 `prm.FF`）生成匹配对
  3. 调用 `calcFI` 计算特征重要性

- **`calcFI(object, prm)`**：计算特征重要性
  1. 对每个匹配对，计算两点特征差的 RMS
  2. 将差值与阈值比较，生成二进制特征矩阵 `PF`
  3. 对每维特征，计算其与匹配标签的**皮尔逊相关系数绝对值**作为重要性

- **`generateTask(object, prm)`**：生成优化任务
  1. 根据 `FI` 对每个描述子内部的特征排序
  2. 为重要的描述子（第 1、3 个）选取前 90% 的特征构成**有前景区域（Promise Area）**
  3. 生成两个任务：
     - **任务1**：仅在有前景区域内搜索（缩小搜索空间）
     - **任务2**：在全部特征空间内搜索（保证全局探索）

- **`generateTask2(object, prm, proportion)`**：参数化版任务生成
  - 与 `generateTask` 类似，但 `proportion` 参数控制主要描述子的选取比例

- **`updateModel(object, idv, prm)`**：更新代理模型
  - 使用当前最优个体生成新的匹配对，加入数据库

### 3.3 TASK 类（`Public/TASK.m`）

轻量级任务定义类。

| 属性 | 说明 |
|------|------|
| `dim` | 任务维度（等于总特征维度） |
| `space` | 搜索空间向量（1 × dim），`1` 表示该维度参与搜索，`0` 表示固定为 0 |

### 3.4 INDIVIDUAL 类（`Public/INDIVIDUAL.m`）

种群管理类，封装遗传算法的所有操作。

| 属性 | 说明 |
|------|------|
| `pop` | 种群大小 |
| `idvs` | 个体矩阵（pop × dim），二进制编码，`1` 表示选择该维特征 |
| `fits` | 适应度向量（pop × 1） |
| `task` | 所属任务（TASK 对象） |
| `pc` | 杂交概率 |
| `pm` | 变异概率 |
| `elite_num` | 精英个体数 |
| `prm` | 参数引用 |

**核心方法**：

- **`initINDIVIDUAL(object, prm, pop, task, pc, pm, elite_num, W)`**：初始化种群
  - 生成随机个体，根据权重向量 `W` 控制每维特征被选中的概率
  - 不在搜索空间内的维度固定为 0
  - 使用 `eva4` 评估初始适应度

- **`cross(Individual, pairs)`**：**双点交叉**
  - 对轮盘赌选出的父代对，以概率 `pc` 进行交叉
  - 交换长度固定为 20 个基因位

- **`mutation(Individual, o1)`**：**位翻转变异**
  - 对每个基因位，以概率 `pm` 进行翻转（0↔1）
  - 不在搜索空间内的位不参与变异

- **`generatePairs(Individual)`**：**轮盘赌选择父代对**
  - 根据适应度进行轮盘赌选择，生成 pop/2 对父代

- **`select(Individual, all, fits)`**：**精英保留 + 轮盘赌选择**
  - 前 `elite_num` 个位置保留最优个体
  - 其余位置通过轮盘赌选择

- **`generateNewIndividualFromOld(object, ntask, W)`**：**任务空间更新**
  - 当任务的搜索空间发生变化时，保留新旧任务共有的特征
  - 对新增的搜索维度，根据权重 `W` 随机初始化

## 4. 主入口脚本

### 4.1 `main.m` —— 完整对比实验

执行流程：
1. **参数初始化**：创建 `PARAMETER` 和 `DB` 对象
2. **数据预处理**：生成特征标准差，调用 `init` 处理数据
3. **运行 6 种方法**，依次为：
   - **ReliefF**（`calcRelieff`）：基于近邻样本的特征权重算法
   - **皮尔逊相关系数**（`calcperson`）：基于线性相关性的过滤式方法
   - **互信息**（`calcMI`）：基于信息论的过滤式方法
   - **CUS-PSO**（`calcCUS`）：基于代理模型的粒子群优化
   - **MF-CSO**（`calcMFCSO`）：多目标柯西鲨鱼优化器
   - **AEMT**（`method`）：本文提出的进化多任务方法
4. **效果测试**：对每种方法选出的特征子集，调用 `test` 计算配准精度
5. **输出**：匹配分数矩阵 `scoress` 和匹配数矩阵 `matchnums`

### 4.2 `PSTMain.m` —— 参数敏感性测试

测试两个关键参数对结果的影响：
- **`proportion`**（0.4 ~ 1.0，步长 0.1）：主要描述子的特征选取比例
- **`G`**（5 ~ 30，步长 5）：进化代数

对每组参数组合，运行 `PSTmethod` 并记录配准精度，输出为二维矩阵。

## 5. 核心优化算法（Algorithm/）

### 5.1 `method.m` —— AEMT（进化多任务特征选择）

这是本项目提出的**核心算法**。

**算法参数**：
| 参数 | 默认值 | 说明 |
|------|--------|------|
| `TE` | 1 | 真实评估轮数（外层循环） |
| `ite` | 10 | 每轮迭代次数（内层循环） |
| `pop` | 10 | 种群大小 |
| `pc` | 0.1 | 杂交概率 |
| `pm` | 20/dim | 变异概率 |
| `elite_num` | 1 | 精英保留数 |
| `W1` | `[0.8, 0.5, 1, 0.5]` | 任务1的初始化权重（按描述子分配） |
| `W2` | `[0.8, 0.5, 1, 0.5]` | 任务2的初始化权重 |

**算法流程**：

```
输入: prm (参数), d (数据库)
输出: best (最优特征子集), time (运行时间)

1. 生成任务: tasks = generateTask(d, prm)
   - tasks(1): 有前景区域搜索空间
   - tasks(2): 全部特征空间

2. 初始化种群:
   - idvs(1): 任务1的种群，权重W1
   - idvs(2): 任务2的种群，权重W2

3. 外层循环 (te < TE):
   3.1 内层循环 (it = 1:ite):
       a) 杂交变异: 对每个任务的种群做双点交叉和位翻转变异
       b) 知识迁移:
          - 对全局任务(tn): 接收其他任务的最优个体（直接迁移 + 片段替换）
          - 对局部任务: 从全局任务的最优个体中提取对应维度
       c) 评估与选择: 合并父代与子代，精英保留+轮盘赌选择
       d) 记录最优适应度

   3.2 代理模型更新 (te < TE):
       a) 使用当前最优个体更新数据库匹配对
       b) 重新计算特征重要性
       c) 重新生成任务
       d) 更新种群的搜索空间

4. 返回任务1的最优个体作为结果
```

**知识迁移机制（核心创新点）**：
- **局部→全局**：将局部搜索任务的最优个体直接注入全局任务种群
- **局部→全局（片段替换）**：用局部最优个体的对应片段替换全局最优个体的相应位置
- **全局→局部**：将全局最优个体在局部搜索空间内的部分提取出来注入局部种群

### 5.2 `PSTmethod.m` —— 参数敏感性测试版

与 `method.m` 结构几乎相同，主要区别：
- 使用 `generateTask2` 代替 `generateTask`，支持 `proportion` 参数
- 迭代次数 `ite` 由外部参数 `G` 控制
- `W1` 设为 `[0.8, 0.6, 1, 1]`（稍有不同）

### 5.3 `nomultitask.m` —— 无多任务对比版

消融实验用，去掉多任务机制的版本：
- 仅使用单一任务（全部特征空间）
- 无知识迁移
- 用于验证多任务策略的有效性

## 6. 操作与工具函数（Operators/）

### 6.1 数据处理函数

#### `init.m` —— 数据预处理
- 对每对点云，使用全部特征进行匹配
- 调用 `pcmatchfeatures` 进行特征匹配
- 使用 RANSAC 进行鲁棒变换估计
- 保存处理结果

#### `generatePair.m` —— 匹配对生成
- 输入：特征子集 `X`（可以是多个）和参数 `prm`
- 对每对有足够重叠率的点云组合：
  1. 根据 `X` 选择特征子集
  2. 使用 `pcmatchfeatures` 进行特征匹配
  3. 计算变换后的点距离，标记正确匹配（距离 < 阈值）和错误匹配
- 输出：匹配对矩阵 `[点云i, 点云j, 点i索引, 点j索引, 标签(1/0)]`

#### `generateSTD.m` —— 特征标准差计算
- 对所有点云的每个特征维度计算标准差
- 用于特征归一化

#### `getAllF.m` —— 特征加载与归一化
- 加载 FPFH、SHOT、SpinImage、ROPS 四种描述子
- 去除全零列
- 按列归一化到 [0, 1]
- 拼接为统一的特征矩阵

#### `getAllRadiusF.m` —— 多尺度特征加载
- 加载不同搜索半径下计算的特征
- 支持多尺度匹配实验

#### `calcOverlap.m` —— 点云重叠率计算
- 将点云 p1 变换到 p2 的坐标系
- 统计 p1 中与 p2 中任意点距离小于阈值的点的比例

#### `estimateTransform.m` —— SVD 刚体变换估计
- 基于 SVD 分解的经典三维点集配准算法
- 输入：两组对应点
- 输出：旋转矩阵 R 和平移向量 t

#### `findpairs.m` —— 几何一致性匹配对查找
- 基于刚性变换的几何一致性约束
- 检查匹配对之间的距离一致性

#### `cal_yuzhi.m` —— 阈值计算
- 根据点云的空间分布自动计算匹配阈值

### 6.2 评估函数

#### `eva4.m` —— 主评估函数
这是整个系统最核心的评估函数，被遗传算法反复调用。

**输入**：个体矩阵 `idvs`（每行是一个二进制特征子集）和参数 `prm`

**评估流程**（对每个个体）：
1. 选择指定维度的特征子集
2. 对多对点云进行特征匹配（`pcmatchfeatures`）
3. 使用真值变换将匹配点对齐
4. 计算对齐后的点距离
5. 统计正确匹配比例（距离 < 阈值）
6. **适应度 = 所有点云对的平均正确匹配率**

**关键细节**：
- 使用 `prm.p`（代理模型的下采样点云）或 `prm.pp`（真实点云）进行评估
- 匹配方法支持穷举搜索（`Exhaustive`），匹配阈值 0.95，拒绝比例 0.95

#### `eva2.m` —— 简化评估函数
- 不使用直接的点对应关系
- 基于 RANSAC 变换估计后的误差进行评估
- 计算变换误差和旋转误差

#### `test.m` —— 最终效果测试
- 与 `eva4` 类似但更完整
- 用于最终对比实验，返回每对点云的匹配分数和匹配数量

### 6.3 可视化函数

#### `dopairpicture.m` —— 匹配对可视化
- 将两个点云变换到同一坐标系
- 使用 `pcshowpair` 和 `pcshow` 显示匹配效果

#### `getCorrAndRegFig.m` —— 多尺度匹配可视化
- 测试不同尺度下的特征匹配效果
- 生成对应关系图和配准结果图

#### `error_map/` —— 误差热力图
- `error_map.m`：生成配准误差的热力图可视化
- 支持自定义颜色映射和标注

## 7. 基线对比算法

### 7.1 过滤式方法

#### ReliefF（`calcRelieff.m`）
- 经典的 ReliefF 算法
- 计算每维特征的权重
- 选择权重大于 0 的特征
- 调用 `Operators/CUS-SPSO/fsReliefF.m`（使用 Weka 库）

#### 皮尔逊相关系数（`calcperson.m`）
- 计算每维特征与匹配标签的皮尔逊相关系数
- 按描述子分组，每组选取前 N 个最相关的特征
- 选取比例根据描述子维度自适应调整

#### 互信息（`calcMI.m`）
- 使用 `MutualInfo4.m` 计算每维特征与标签的互信息
- 选取互信息值最大的特征
- 选取策略类似皮尔逊方法

#### PCC + MI 联合（`calcPCCAndMI.m`）
- 同时使用皮尔逊相关系数和互信息
- 对两种指标的排序取并集

### 7.2 包裹式方法

#### CUS-PSO（`calcCUS.m` + `CUS-SPSO/`）
基于代理模型的粒子群优化特征选择。

**核心组件**：
- `CUSmain.m`：主算法，使用 ReliefF 初始化 + KNN 代理模型
- `Particle.m`：粒子类（位置、速度、个人最优、全局最优）
- `SurrogatePSO.m`：使用 KNN 代理模型的 PSO
- `SurrogateKNN.m`：KNN 代理适应度评估
- `fitness.m`：真实适应度函数（5 折交叉验证 + 特征数惩罚）

**流程**：
1. 使用 ReliefF 生成初始解
2. 使用 KNN 构建代理模型
3. 在代理模型上运行 PSO 搜索
4. 定期用真实评估更新代理模型

#### MF-CSO（`calcMFCSO.m` + `MF_CSO/`）
多目标多因子柯西鲨鱼优化器。

**核心组件**：
- `MFmain.m`：主算法入口
- `multitask.m`：多任务竞争学习 PSO
- `divide_four_task.m`：将特征分为 4 个任务（使用膝点检测）
- `fitness.m`：多目标适应度（错误率 + 特征数 + 距离）
- `MOEAD.m`：多目标进化分解算法
- `nsga.m`：NSGA-II 非支配排序

**特点**：
- 使用柯西分布（`cauchy/` 目录）进行变异操作
- 支持多目标优化
- 基于特征重要性进行任务划分

#### SS-PSO（`calcSS.m` + `SS-PSO/`）
子群粒子群优化特征选择。

**核心组件**：
- `SSmain.m`：主算法
- `grouping.m`：基于对称不确定性（SU）和层次聚类的特征分组
- `ppso1.m`：带任务分解的 PSO 更新
- `evaluation.m`：使用 3NN + 5 折 CV 评估
- `sub_model_0/1.m`：最小二乘代理模型

## 8. 评估函数

系统中存在多层评估体系：

| 函数 | 用途 | 评估对象 | 计算成本 |
|------|------|---------|---------|
| `eva4` | 进化搜索中的适应度评估 | 下采样点云 `prm.p` | 低（代理模型） |
| `eva4`（真实评估） | 代理模型更新时 | 原始点云 `prm.pp` | 高 |
| `test` | 最终效果测试 | 原始点云 `prm.pp` | 高 |
| `eva2` | 基于 RANSAC 的评估 | 原始点云 | 高 |

**适应度计算核心公式**：

```
适应度 = Σ(正确匹配数 / 总匹配数) / 点云对数
```

其中"正确匹配"定义为：经真值变换对齐后，两匹配点距离 < `yuzhi`。

## 10. 关键设计思想

### 10.1 多任务优化框架

将特征选择问题分解为两个关联任务：
1. **局部搜索任务**：在根据特征重要性筛选出的"有前景区域"内搜索，加速收敛
2. **全局搜索任务**：在全部特征空间内搜索，避免遗漏

两个任务通过**知识迁移**相互促进：
- 局部任务的精细解帮助全局任务快速定位好区域
- 全局任务的多样性帮助局部任务跳出局部最优

### 10.2 代理模型策略

使用**下采样点云**作为代理模型：
- `prm.p`（下采样两次）：用于进化搜索中的快速评估
- `prm.pp`（下采样一次）：用于真实评估和最终测试

这种策略大幅降低了评估成本，因为点云配准的计算复杂度与点数成正比。

### 10.3 动态任务空间更新

通过代理模型更新机制，搜索空间在进化过程中动态调整：
1. 每轮真实评估后，将当前最优个体用于生成新匹配对
2. 新匹配对更新特征重要性
3. 根据更新后的重要性重新划分搜索空间
4. 种群继承旧空间的基因，并对新增空间随机初始化

### 10.4 特征重要性驱动的初始化

种群初始化不是完全随机的，而是受权重向量 `W` 控制：
- 不同描述子有不同的初始选中概率（如 SpinImage 为 1.0，SHOT 为 0.5）
- 这反映了先验知识：某些描述子可能整体质量更高

### 10.5 二进制编码与遗传操作

- **编码**：每个个体是 dim 维的二进制向量，`1` 表示选择、`0` 表示不选
- **交叉**：固定长度（20 位）的双点交叉
- **变异**：独立位翻转，概率 `20/dim`（期望每个个体翻转约 20 位）
- **选择**：精英保留（1 个）+ 轮盘赌选择
